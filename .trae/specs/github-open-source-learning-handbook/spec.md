# GitHub 开源学习手册与安全加固 Spec

## Why
当前项目已具备完整链路能力，但若要作为 GitHub 开源学习项目，需要“可读、可跑、可验证、可扩展”，并在开源前完成基础安全与质量加固，避免泄露密钥、错误默认配置与脆弱实现。

## What Changes
- 新增一份面向读者的“学习手册”，按模块与数据流逐步引导读者读懂系统，并给出可复现的本地运行与调试路径。
- 新增开源前安全治理：清理仓库中的明文密钥/敏感配置，提供模板与说明，补齐最小权限与默认安全姿势。
- 进行代码审计与缺陷修复：修复实际可触发的错误、边界条件与健壮性问题，并补齐可回归的验证用例。
- 进行冗余代码清理与小步重构：删除未使用变量/重复逻辑、统一错误处理与配置读取路径，降低认知复杂度。
- 新增开源友好的工程化设施：基础 CI（构建 + 关键 smoketest）、贡献指南与目录结构说明。

## Impact
- Affected specs: 学习手册、开源发布、运行与调试、配置管理、SSE 流式、Embedding/Qdrant/RAG、记忆索引、面试业务闭环。
- Affected code: 配置加载与示例文件、路由与鉴权中间件、记忆索引与 Qdrant 写入、测试/自检可执行、README 与新增文档、CI 配置。

## ADDED Requirements
### Requirement: 学习手册
系统 SHALL 提供一份完整学习手册，覆盖从“跑起来”到“读懂数据流”再到“动手改造”的路径。

#### Scenario: 新读者从零开始
- **WHEN** 读者首次克隆仓库并按手册操作
- **THEN** 能在本地成功启动服务、跑通关键 smoketest、并理解模块边界与数据流（鉴权→会话→SSE→落库→索引→检索→注入→面试业务）

### Requirement: 开源前安全治理
系统 SHALL 不在仓库内存放可用的明文密钥与访问凭证，并提供配置模板与最小权限建议。

#### Scenario: 仓库公开时不泄露凭证
- **WHEN** 仓库被公开发布
- **THEN** 仓库中不存在可直接用于访问外部服务的 key/token/URL 组合，且默认配置不会以不安全方式运行于生产环境

### Requirement: 质量与可回归
系统 SHALL 提供可回归的构建与验证路径（本地与 CI），确保每次变更后关键链路仍可跑通。

#### Scenario: 变更后回归验证
- **WHEN** 提交修改并触发 CI
- **THEN** 构建成功，关键自检（至少：qdrant/embedding、memory_index、SSE hold、核心 API 端到端脚本）通过

## MODIFIED Requirements
### Requirement: 默认配置与可诊断性
系统 SHALL 对缺失配置给出明确提示（不打印密钥明文），并在关键失败场景输出可定位的错误信息，便于读者学习与排障。

## REMOVED Requirements
无
